WELCOME to the Boncuklu excavation module, prepared by FAIMS for collection of excavation data at the Neolithic site of Boncuklu in Konya Province, Turkey (http://boncuklu.org), and designed for use on the FAIMS mobile platform available on Android tablets from the Google Play Store. This module is the main medium of data collection on the Boncuklu project allowing the integrated collation of text descriptions (using defined terms), reference photographs, sketches and video/audio records of trenches, features, contexts and finds at the excavation trench.

USERS must be registered with the module administrator (a.fairbairn@uq.edu.au) and login with their User ID at the start of every session.

STRUCTURE. Data are entered into a series of fields via text input, dropdown menus and use of tablet tools such as cameras and microphones, with sketches added from a drawing app. Data fields are organised into a series of logical tabs and tab groups which allow recording of:

  • Contexts: individual archaeological entities defined during excavation including deposits, cuts, structures, human skeletons and artefact clusters;
  • Features: being groups of associated contexts which together form distinct elements of the site, such as burials (consisting of cuts, skeletons and deposits, on occasion with artefacts and artefact clusters), pits (consisting of cuts and deposits) and hearths (consisting of cuts, structural elements and deposits (fills)). Graves are special types of features, as are buildings;
  • Artefacts: as small finds/groundstone items found in contexts, and as artefact clusters, a specific kind of context consisting of a group of associated small finds or groundstone items;
  • Samples: quantities of soil or other material collected for specific scientific analysis from contexts;
  • Photographic log entries: records of photographs taken as part of the excavation archive.
  • Diary entries: Records of events during the excavation at specific excavation trenches.

SAVING AND BACKUP: Data are saved automatically as they are entered. The Boncuklu Excavation Module allows syncing to a local or remote server which collates data from multiple devices.

BEWARE: context and feature codes, sample numbers and small find/groundstone numbers must be assigned from the Central Register by trench supervisors and/or the directors.

Using the Boncuklu excavation module:

  1. Open the FAIMS mobile platform
  2. If you are using the module for the first time:
    1. Press the List Modules button
    2. Go to Settings (on Galaxy 8 tablets it is found by pressing the menu button, bottom left)
    3. Choose autodiscover server
    4. Select the latest version of the Boncuklu Excavation Module, install and open.
  3. Module Home Page/Login in screen: To enter the module you have to pick your name from the dropdown menu, select the excavation which you are working in and select the relevant device number every time to you login. Your name should be on the user list, if not contact the module administrator (a.fairbairn@uq.edu.au). Once logged in the Navigation Screen will appear (see 4).The Help Tab and Module Guide Button will open up this help item! On the login screen explore two important features which can be used at any time:
    1. Sidebar: hold your finger or stylus down on the left margin of the screen and swipe across it. A bar will appear which allows you to navigate through the module and will complement the tab menus on each tab group (contexts, features etc);
    2. Action bar: this is found on the Galaxy tablets by pressing the menu button (bottom left). It allows you to start and stop syncing with the server and to manage GPS devices.
  4. Navigation (Control) screen: This screen offers two choices via the Context and Records Tabs and is the navigational origin point for all module operations once logged in.
    1. Context tab: buttons allow you to create new:
      1. Context records: the description of all individual archaeological entities, forming the core of our descriptive system, and their relationships to other contexts;
      2. Feature records: groups of contexts forming discrete features such as graves, hearths etc, including the relationship of features to each other;
      3. Diary entries: free form notes, photos and audio/video descriptions to be added at any time to document the progress of the excavation;
      4. In addition, values are set here for team member initials when contexts/features are being excavated by multiple diggers. In addition starting values for the context, feature and sample numbers can be added here.
    2. Records tab: this is the search engine for selected data fields and allows you to search and open records entered into the module by clicking on them.
      1. The searchable entities (contexts, features etc) are listed at the bottom of the screen. To refine the list enter text into the “Search term” box or use the dropdown menus Entity Type and Trench.
      2. When you find a record click on it to open and edit/view.
      3. The entity list has a very specific format which lists the trench and then whether the entity is a context (Cxtx) or feature (Feat), then the code/number and other information. This makes the entity identifiers easier to understand.

Adding Context and Feature records

  1. Use the “Create new context” and “New Feature” buttons to add new records
  2. All records autosave and no buttons need to be pressed.
  3. As in the 2014 version the data for each context/feature is added in the tabs working from left to right and from top to bottom of the screens. Pressing buttons on some screens (e.g. “Fill out context details”) opens up new tabs. Use the sidebar and, secondarily, the back button to navigate between tabs.
  4. The “Features” tab (Feat) in Contexts allows you to add a context to a specified feature and open feature records. The Contexts Tab in features allows you to open context records and assign them to that feature.
  5. Descriptions of fields in the Boncuklu Excavation Module should follow the detailed Recording Guide where dropdown menus are not available.
  6. Essential fields that should be filled in are identified in red. Validate buttons and tabs (abbrev. to Val in the context tab group) allow you to see which of these fields has not been completed for a particular tab/tab group. When all of the red entries have been completed a message will say that everything has been entered. Note that this only applies to the specified core data.
  7. Annotation, certainly and information fields are available via the little icons in the top right of each field. Add text in the annotation fields when additional data are needed which cannot be entered into the main field. When any text is added to the annotation field the pencil icon turns blue.
  8. Relationships between features and contexts require you to relate different contexts and features selected from a list (opened by pressing the “Populate target content” button”), using the range of specified terms in the dropdown menu. Please note that all possible relationships for both contexts and features are present in the lists and correct assignment requires knowledge of the Boncuklu recording system. If you wish to specify relationships to a context or feature excavated in previous years you can add a placeholder for those contexts by pressing the “Create Legacy” button. Hit that, add the relevant context or feature code from earlier years and you can add it to the relationships.
  9. Photographs, audio and video records can be added to the records directly using the buttons distributed throughout the module (in contexts see the Add tab) which open cameras and microphones on your device.
  10. References to archive photographs can be added using the “Add Photographic Log” tab and buttons. The Photo ID number in these tabs refers to the standard site photographic log number.

Adding Sketches/Drawings to records in FAIMS

You cannot prepare sketches within FAIMS but need to use an external app to do so and one that downloads files in a generically readable format. We use Sketchbook Pro, which downloads images in tiff format. This should be pre-loaded on the tablets. These instructions are specifically for Autodesk Sketchbook run on Samsung Galaxy tablets. To view the tiff files it generates on the tablet and in FAIMS via “View attached files” you also need to download a tiff friendly app such as Fast Image Viewer.

  1. To draw a simple sketch:
    1. Open the app and run through the tutorial (choose the info/settings icon - it is a cog, second from left) to become familiar with how to use the tools. You can choose various nibs, colours and styles of pen/brush etc and also erasers;
    2. Open a new sketch by pressing the icon containing four squares in the top left and selecting from the menu. Select default resolution or go to custom for higher resolution (for plans etc);
    3. To view the layers, each of which can be drawn upon separately, select the top right icon (looks like 3 sheets of paper superimposed). The layers in your drawing are visible and can be added by pressing the + button and deleted by pressing down and moving to the bin (becomes visible when the layer is held down). Note that the layers are shown in order with the uppermost one at the top. The active layer - that is the one on which you can draw etc - is highlighted in blue
    4. Add a layer and play with the layers to see how this works.
    5. Now draw your sketch on the layer and always write the site code and year (e.g. BK15, trench number) as well as the context code and short description. This can be made very small by zooming in and out (press with two fingers and move together or apart).
    6. Save your creation by going to the ‘four boxes’ icon in top left corner and pressing save.
    7. The app currently saves the file as .tiff with a random number as filename in its own ‘gallery’. You currently cannot change these defaults which is why you must write the context info on the sketch.
  2. To import a picture and sketch over it
    1. Tap on one of the layers (second one is best) and see options to ‘import picture’ from camera (takes a new picture) or from a gallery (uses an old picture), etc.
    2. Make sure that your topmost/selected layer is blank and draw on it, making sure to include the site code, context etc as above
    3. Save as above.
    4. You can thus import pro-forma sheets for sketching, such as the BK Skeleton Sheet (should be pre-loaded in your Tablet's Gallery) or take a picture of what you want to sketch and trace over it, retaining or deleting the picture layer as you see fit.
  3. To attach the file to a FAIMS record
    1. Navigate to the appropriate tab within your module. Press the “Attach a sketch/file” button.
    2. In the ‘Choose files’ dialogue navigate to ‘Autodesk’. This is usually under  ‘emulated>legacy’, but may be elsewhere on your device.
    3. If you are importing the sketch immediately select the most recent .tiff file.
    4. You can order items in the folder by pressing the Menu button (bottom left) and selecting ‘Sort by Date’.
    5. Tapping the file will attach it to the FAIMS Record.
  4. To view the file click on ‘View attached files’ and tap the filename you want to view. Note that to see tiff files generated by Autodesk Sketchbook you will need to install a tiff capable viewer such as Fast Image Viewer.
